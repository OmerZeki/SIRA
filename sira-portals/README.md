# SIRA Portals Service

A standalone Node.js/Express microservice deployed on Render that powers the **SIRA Portal Hub** feature.

## What it does

| Feature | Endpoint | Description |
|---|---|---|
| Health check | `GET /health` | Returns service status + portal list |
| Portal registry | `GET /portals` | Returns full portal metadata |
| Reverse proxy | `GET /proxy?url=<encoded>` | Strips `X-Frame-Options` so any portal can be embedded in a SIRA iframe |
| Screenshot | `POST /api/screenshot` | Playwright headless screenshot of any URL |
| LMIS automation | `POST /api/automate/lmis` | Logs into E-LMIS with agency credentials |
| Musaned automation | `POST /api/automate/musaned` | Logs into Tawtheeq Musaned with agency credentials |
| Enjaz/MOFA | `POST /api/automate/enjaz` | Opens MOFA portal (manual required — B2B agreement needed) |

## Deploy to Render

1. Push the `sira-portals/` folder as its own Git repo (or create it as a sub-directory service on Render).
2. In the Render dashboard create a **Web Service** → Node → set the build command to:
   ```
   npm install && npx playwright install chromium --with-deps
   ```
3. Set start command to `node server.js`.
4. Add the environment variables from `.env.example`.
5. The `PORTAL_API_SECRET` will be auto-generated by Render if you use `render.yaml`.

## Local development

```bash
cp .env.example .env
# Fill in PORTAL_API_SECRET etc.
npm install
npx playwright install chromium
node server.js
# → Listening on http://localhost:4000
```

## Security

- All automation endpoints require `Authorization: Bearer <PORTAL_API_SECRET>`.
- CORS is restricted to `SIRA_APP_ORIGIN`.
- The `/proxy` endpoint blocks private/loopback addresses.
